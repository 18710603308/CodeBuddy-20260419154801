## 📋 前置准备

### 1. 注册阿里云账号并登录
- 访问 [阿里云官网](https://www.aliyun.com/)
- 登录你的账号

### 2. 配置ACR镜像仓库（已完成准备）

**步骤1**: 访问 [容器镜像服务 ACR 控制台](https://cr.console.aliyun.com/)

**步骤2**: 创建个人版实例
- 在左侧菜单点击 **实例列表**
- 点击 **创建个人版实例**
- 选择区域（建议选择杭州 `cn-hangzhou`）
- 点击 **立即创建**

**步骤3**: 创建命名空间
- 在左侧菜单点击 **命名空间**
- 点击 **创建命名空间**
- 名称填写：`devtools-hub`
- 点击 **确定**

**步骤4**: 创建镜像仓库
- 在左侧菜单点击 **镜像仓库**
- 点击 **创建镜像仓库**
- 填写信息：
  - 仓库名称：`frontend`
  - 命名空间：选择刚才创建的 `devtools-hub`
  - 仓库类型：公开（或私有）
  - 点击 **确定**

**步骤5**: 获取访问凭证
- 在左侧菜单点击 **访问凭证**
- 设置固定密码（记录用户名和密码，后续会用到）

## 🚀 步骤二：关联代码仓库

### 方式A：关联GitHub仓库

1. 在项目首页点击 **代码源** → **新建代码源**
2. 选择 **GitHub**
3. 点击 **去授权**，登录GitHub并授权阿里云DevOps访问你的仓库

### 方式B：关联其他代码仓库

如果你的代码在GitLab/Gitee等平台：

3. 点击 **确定**

## 🚀 步骤三：创建流水线

### 3.1 创建流水线

1. 在项目首页点击 **流水线** → **新建流水线**
2. 选择 **空白模板** → 点击 **确定**
3. 输入流水线名称：`前端自动化部署`
4. 点击 **保存**

### 3.2 添加构建阶段

**阶段配置**:
- 阶段名称：`构建镜像`
- 执行方式：`顺序执行`

**添加任务1：拉取代码**
- 点击 **添加任务**
- 搜索并选择 **拉取代码**
- 配置：
  - 代码源：选择你关联的仓库
  - 分支：`main`
  - 点击 **确定**

**添加任务2：构建Docker镜像**
- 点击 **添加任务**
- 搜索并选择 **Docker 构建**
- 配置：
  - 任务名称：`构建前端镜像`
  - Dockerfile路径：`Dockerfile.frontend`
  - 镜像标签：
    ```
    registry.cn-hangzhou.aliyuncs.com/devtools-hub/frontend:${DATETIME}
    ```
  - 点击 **确定**

**添加任务3：推送到ACR**
- 点击 **添加任务**
- 搜索并选择 **Docker 推送**
- 配置：
  - 任务名称：`推送到ACR`
  - 镜像标签：（与构建任务相同）
    ```
    registry.cn-hangzhou.aliyuncs.com/devtools-hub/frontend:${DATETIME}
    ```
  - Docker仓库连接：
    - 点击 **添加连接**
    - 连接类型：`Docker Registry`
    - Registry地址：`registry.cn-hangzhou.aliyuncs.com`
    - 用户名：你的阿里云账号
    - 密码：你的ACR访问凭证密码
    - 点击 **确定**
  - 选择刚创建的连接
  - 点击 **确定**

### 3.3 添加部署阶段

**点击 "+" 添加新阶段**

**阶段配置**:
- 阶段名称：`部署到服务器`
- 执行方式：`顺序执行`

**添加任务：SSH执行**
- 点击 **添加任务**
- 搜索并选择 **SSH执行**
- 配置：
  - 任务名称：`部署应用`
  - 选择主机：
    - 点击 **添加主机**
    - 主机名称：`腾讯云服务器`
    - IP地址：`110.42.247.238`
    - 端口：`22`
    - 用户名：`root`
    - 认证方式：选择 **密码**
    - 密码：你的服务器密码
    - 点击 **确定**
  - 在 **执行命令** 输入框中粘贴以下脚本：

```bash
#!/bin/bash
set -e

# 配置
ACR_REGISTRY="registry.cn-hangzhou.aliyuncs.com"
ACR_NAMESPACE="devtools-hub"
ACR_REPO="frontend"
IMAGE_TAG="${DATETIME}"
APP_NAME="devtools-hub"
CONTAINER_NAME="${APP_NAME}-app"

echo "=== 开始部署 ==="
echo "镜像: ${ACR_REGISTRY}/${ACR_NAMESPACE}/${ACR_REPO}:${IMAGE_TAG}"

# 1. 登录ACR
echo "1/4 登录ACR..."
docker login --username=你的阿里云账号 --password=你的ACR密码 ${ACR_REGISTRY}

# 2. 拉取镜像
echo "2/4 拉取镜像..."
docker pull ${ACR_REGISTRY}/${ACR_NAMESPACE}/${ACR_REPO}:${IMAGE_TAG}

# 3. 停止并删除旧容器
echo "3/4 清理旧容器..."
if docker ps -a | grep -q ${CONTAINER_NAME}; then
    docker stop ${CONTAINER_NAME}
    docker rm ${CONTAINER_NAME}
fi

# 4. 启动新容器
echo "4/4 启动新容器..."
docker run -d \
    --name ${CONTAINER_NAME} \
    --restart=always \
    -p 8081:80 \
    ${ACR_REGISTRY}/${ACR_NAMESPACE}/${ACR_REPO}:${IMAGE_TAG}

# 验证
sleep 3
if docker ps | grep -q ${CONTAINER_NAME}; then
    echo "✅ 部署成功!"
    docker ps --filter name=${CONTAINER_NAME}
    echo ""
    echo "访问地址: http://110.42.247.238:8081"
else
    echo "❌ 部署失败!"
    docker logs ${CONTAINER_NAME}
    exit 1
fi
```

### 3.4 配置变量

2. 添加变量：

## 🔧 常见问题排查

### Q1: Docker构建失败

**可能原因**:
- Dockerfile路径错误
- 代码中缺少依赖

**解决方法**:
- 检查 `Dockerfile.frontend` 是否存在
- 确保前端项目已正确配置

### Q2: 推送ACR失败

**可能原因**:
- 仓库地址错误
- 认证信息错误

**解决方法**:
- 检查Registry地址是否正确
- 确认ACR凭证用户名和密码正确

### Q3: 服务器部署失败

**可能原因**:
- 服务器网络无法访问ACR
- Docker服务未运行
- 端口被占用

**解决方法**:
- 在服务器上测试网络连接：`curl registry.cn-hangzhou.aliyuncs.com`
- 检查Docker状态：`systemctl status docker`
- 检查端口占用：`netstat -tlnp | grep :8081`

### Q4: 流水线不自动触发

**可能原因**:
- 触发配置未启用
- 代码源连接失效

**解决方法**:
- 检查流水线的 **触发** 配置
- 重新授权代码源连接
